# Skills Triggering in Action — 28 技能批量修復實錄

> **日期：** 2026-06-09  
> **背景：** 在設計 Agent Repair 技能時，發現方法論技能全部未被調用。根因診斷後確認：28 個技能的 description 缺少中文觸發關鍵詞。

---

## 🔬 三斷點診斷

### 技術故障鏈

```
用戶任務（中文）
    │
    ▼
[斷點 1] Description 關鍵詞匹配失效
    │  28 個技能無中文關鍵詞 → 對中文任務完全隱形
    │  包括所有方法論技能：idea-refine、software-architect、caveman 等
    │
    ▼
[斷點 2] 模型不認為需要調用技能
    │  模型把「我知道怎麼做」等同於「我不需要技能」
    │  技能的價值是結構化流程，不是填補知識空白
    │
    ▼
[斷點 3] R17 沒有技術攔截點
    │  規則存在於文件中，但沒有程式碼強制執行
    │  依賴模型「自律」——在長對話中不可靠
    │
    ▼
結果：直接開工 → 跳過所有方法論技能
```

### 根因

6 個核心方法論技能中，4 個的 description 只有英文或格式損壞，對中文任務完全隱形。合計 28 個技能受到影響。

---

## 🩹 修復方案

### 斷點 1：批量中文關鍵詞注入 ✅

使用 `skills-triggering.py` 批量修復 28 個技能的 description：

| 類別 | 數量 | 範例 |
|------|------|------|
| 方法論技能 | 15 | software-architect → `架構設計 系統設計 軟件架構` |
| 系統/運維 | 7 | self-improving-agent → `自我改進 記錄教訓 錯誤學習` |
| 格式損壞修復 | 6 | agent-audit、claude-code 等 |

### 斷點 2+3：AGENTS.md Step -2 強制閘門 ✅

在 Session 啟動流程中，新增優先級最高的 Step -2：

```
🔴 STEP -2: SKILL-ROUTER PRE-CHECK
收到任何用戶訊息 → 立即：
1. 讀取 skill-router/SKILL.md
2. 分類類別×階段 → 推薦技能
3. 讀取並執行推薦技能
4. 無例外，跳過 = 違反協議
```

同步在 RULES.md 中添加 R0 規則。

---

## 📊 修復效果

| 指標 | 修復前 | 修復後 |
|------|--------|--------|
| 方法論技能中文可見性 | 2/6 (33%) | 6/6 (100%) |
| 總技能中文關鍵詞覆蓋 | ~85% | ~100% |
| 技能調用前置檢查 | 依賴模型自律 | 啟動流程強制 |
