## ① Skills Triggering — 技能觸發

```
請生成一張小紅書風格的教學推廣圖，整體色調溫暖柔和，米白/奶油色底色，深灰色文字，點綴翡翠綠和淡珊瑚色。畫面信息密度中等，留白充足，現代簡約風格。

【頂部標題】
大號粗體文字：「🌐 你的 Agent 技能，對非英語用戶隱形嗎？」
下方副標題（小字灰色）："75% 的 AI Agent 使用者非英語母語 · 95% 開源技能 description 只有英文"

【左上區域 — 問題演示】
一個對話氣泡示意圖：左側是用戶頭像（亞洲面孔），旁邊對話框寫「幫我做個網站系統」（繁體中文，橙色氣泡）。指向中間一個機器人圖標，機器人表情困惑，頭上有問號。旁邊紅色大字 "❌ 0 匹配"。下方小字說明："技能的 description 只有英文——「網站」和「系統」不在英文 description 裡，匹配失敗，技能形同虛設。"

【右上區域 — 解決方案】
同一組對話：用戶說同樣的話，但這次機器人頭上有燈泡亮起（💡），綠色勾號 "✅ 100% 匹配"。機器人旁邊浮現一排技能圖標卡片，每張卡片上寫著中英文關鍵詞："網站設計 · Web Design"、"前端開發 · Frontend"、"UI設計 · UI Design"。下方小字："三層關鍵詞策略——使用者說什麼，description 就包含什麼。"

【中間數據橫幅】
一個圓角膠囊形狀的數據條，白色背景帶淡綠色邊框，水平排列三組數據：
"🔴 修復前：發現率 35% → 匹配率 20%" → 箭頭 → "🟢 修復後：發現率 90% → 匹配率 95%"

【底部區域 — 安裝與行動】

下方一行小字："一行安裝 · 立即生效 · 所有 AI Agent 平台通用"

【最底部頁腳】
"Agentic Infrastructure · 開源 MIT · github.com/Bryan-cmf/agentic-infrastructure"
小紅書風格的細線分隔 + 標籤： "#AIagent #技能觸發 #多語言 #開源"

## ② Skill Router — 技能路由

```
請生成一張小紅書風格的教學推廣圖，整體色調清新專業，淡藍灰/白色底色，深藏藍文字，點綴珊瑚橙和電光藍。畫面佈局清晰分明，信息層次豐富但不過度擁擠。

【頂部標題】
大號粗體文字：「🔀 Agent 有 200+ 個技能，但不知道該用哪個？」
下方副標題（小字）："40% 的 AI 專案因 Agent 協調失敗被取消（Gartner 2026）"

【中央主視覺 — 路由引擎示意圖】
一個從左到右的流程圖風格設計，使用圓角矩形和箭頭連接：

最左側：四個不同顏色的對話氣泡從上方飄入，內容分別是：
💬「幫我做網站」（粉紅） 💬「分析這隻股票」（橙黃） 💬「部署到線上」（淺藍） 💬「修復這個 Bug」（淺綠）

箭頭指向中央一個醒目的六邊形或圓形中樞，發光效果，內部標籤：
「🔀 類別 × 階段」
「矩陣路由」
下方小字："類別檢測 → 階段匹配 → 查表推薦 → 自動展開"

從中樞向右方發散出四條箭頭，每條指向一個具體的技能卡片：
→ 🌐 Frontend Design（匹配：💻代碼 + 🎨設計）
→ 💰 Financial Analyst（匹配：💰金融 + 📊分析）  
→ 🚀 Deploy Vercel（匹配：🛠️日常 + 🚀部署）
→ 🔧 Diagnose（匹配：💻代碼 + 🧪診斷）

【左下區域 — 多階段自動展開示例】
一個小型的流程演示框，淺色背景帶邊框：
"「幫我調研 00058」→"
"Phase 1: ak-hk-stock-dd（搜索）→"
"Phase 2: ak-financial-analyst（分析）→"
"Phase 3: dd-business-report（報告）"
每行前面有一個綠色勾號 ✅

【右下區域 — 效果數據】
三組垂直排列的數據卡片（圓角矩形，白色帶淺藍邊框）：
📊 技能發現率：35% → 90%（+157%）
🎯 錯誤工具使用：減少 80%
⚡ 任務啟動：3-5 回合 → 1 回合

【底部安裝命令】
深色代碼塊："mkdir -p skills/skill-router && curl -sSL [url]/SKILL.md -o skills/skill-router/SKILL.md"

【底部頁腳】同上，標籤："#AIagent #技能路由 #自動化 #開源"
```

## ③ Skill Reporting — 技能匯報

```

【頂部標題】

【主視覺 — Before/After 對比】

頂部標籤 "❌ 沒有 Skill Reporting"

⚡ Debug 加速：看最後一行就知道哪個技能漏了
