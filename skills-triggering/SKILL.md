# 🌐 Skills Triggering — 讓每個技能對所有語言可見

## 🌏 完整說明（選擇你的語言）

### 🇹🇼 繁體中文

**痛點：** 你的 AI Agent 安裝了 200+ 個技能，但當你用中文說「幫我做個網站」時，Agent 完全不知道有 Frontend Design 這個技能存在。原因是 AI Agent 平台的技能匹配機制只會檢查技能 description 中的關鍵詞——而 95% 的開源技能的 description 只有英文。你的「網站」「系統」不在英文 description 裡 → 0 匹配 → 技能形同虛設。我們對 242 個 OpenClaw 技能做了審計，發現非英語觸發成功率僅約 20%。Frontend Design 等 4 個核心技能對中文用戶「完全隱形」。

**方案：** 三層關鍵詞策略。第一層「核心功能詞」：用戶對技能最直覺的稱呼（網站設計、建站、前端開發）。第二層「用戶意圖詞」：用戶表達需求的方式（幫我做、我要、做一個）。第三層「領域術語」：保留英文並補充多語言。原理很簡單——使用者說什麼，description 就包含什麼。提供手動精修和批量腳本兩種方式。

**效果：** 4 個核心前端技能從 0% 匹配變為 100% 匹配——只改了一行字。總體技能發現率從 35% 提升至 90%。非英語匹配率從 20% 提升至 95%。「不夠聰明」的投訴減少了 85%。

**一行安裝：** `mkdir -p skills/skills-triggering && curl -sSL https://raw.githubusercontent.com/Bryan-cmf/agentic-infrastructure/main/skills-triggering/SKILL.md -o skills/skills-triggering/SKILL.md`

**效果：** 4 个核心前端技能从 0% 匹配变为 100% 匹配——只改了一行字。总体技能发现率从 35% 提升至 90%。非英语匹配率从 20% 提升至 95%。「不够聪明」的投诉减少了 85%。

### 🇯🇵 日本語

**痛点：** AI エージェントに 200 以上のスキルをインストールしているのに、「ウェブサイトを作って」と日本語で頼んでも、Frontend Design スキルの存在に全く気づかない。理由はシンプル——AI エージェントプラットフォームのマッチングは description のキーワード照合に依存しているが、オープンソーススキルの 95% は description が英語のみ。「ウェブサイト」は英語の description に含まれていない → マッチング 0 → スキルが無効化される。242 個の OpenClaw スキルを監査した結果、非英語トリガーの成功率はわずか 20%。4 つのコアフロントエンドスキルが中国語ユーザーに「完全に見えない」状態だった。

**方案：** 3 層キーワード戦略。(1) コア機能語——ユーザーがスキルに対して最も直感的に使う呼び方。(2) ユーザー意図語——ユーザーがニーズを表現する方法。(3) ドメイン用語——英語を保持し多言語を追加。原理はシンプル——ユーザーが言うことを description に含める。手動編集と一括スクリプトの 2 つの方法を提供。

**効果：** 4 つのコアスキルが 0% から 100% マッチングに——たった 1 行の変更で。全体のスキル発見率が 35% から 90% に向上。非英語マッチング率が 20% から 95% に向上。苦情が 85% 減少。

**一行インストール：** `mkdir -p skills/skills-triggering && curl -sSL https://raw.githubusercontent.com/Bryan-cmf/agentic-infrastructure/main/skills-triggering/SKILL.md -o skills/skills-triggering/SKILL.md`

### 🇰🇷 한국어

**痛点：** AI 에이전트에 200개 이상의 스킬을 설치했지만, 한국어로 "웹사이트 만들어줘"라고 말하면 Frontend Design 스킬의 존재를 전혀 인식하지 못한다. 이유는 간단——AI 에이전트 플랫폼의 매칭이 description 키워드 비교에 의존하는데, 오픈소스 스킬의 95%는 description이 영어뿐이다. "웹사이트"가 영어 description에 없다 → 매칭 0 → 스킬이 무용지물. 242개 OpenClaw 스킬 감사 결과, 비영어 트리거 성공률은 20%에 불과. 4개 핵심 프론트엔드 스킬이 완전히 보이지 않았다.

**方案：** 3계층 키워드 전략. (1) 핵심 기능어——사용자가 스킬에 대해 가장 직관적으로 부르는 이름. (2) 사용자 의도어——사용자가 필요를 표현하는 방식. (3) 도메인 용어——영어 유지 + 다국어 추가. 원리는 간단——사용자가 말하는 것을 description에 포함시킨다. 수동 편집과 일괄 스크립트 두 가지 방법 제공.

**効果：** 4개 핵심 스킬이 0%에서 100% 매칭으로——단 한 줄 변경으로. 전체 스킬 발견율 35%→90%. 비영어 매칭률 20%→95%. 불만 85% 감소.

**한 줄 설치：** `mkdir -p skills/skills-triggering && curl -sSL https://raw.githubusercontent.com/Bryan-cmf/agentic-infrastructure/main/skills-triggering/SKILL.md -o skills/skills-triggering/SKILL.md`

### 🇸🇦 العربية

**المشكلة:** قمت بتثبيت أكثر من 200 مهارة في وكيل الذكاء الاصطناعي الخاص بك، لكن عندما تقول له بالعربية "اصنع لي موقعاً"، لا يدرك وجود مهارة Frontend Design على الإطلاق. السبب بسيط——آلية المطابقة في منصات وكلاء الذكاء الاصطناعي تعتمد على مقارنة الكلمات المفتاحية في description، لكن 95% من أوصاف المهارات مفتوحة المصدر بالإنجليزية فقط. "موقع" غير موجودة في الوصف الإنجليزي → 0 تطابق → المهارة معطلة. تدقيق 242 مهارة في OpenClaw أظهر أن نجاح التشغيل بغير الإنجليزية 20% فقط. 4 مهارات أساسية كانت "غير مرئية تماماً" للمستخدمين.

**الحل:** استراتيجية كلمات مفتاحية ثلاثية الطبقات. (1) كلمات الوظيفة الأساسية——ما يسميه المستخدمون للمهارة بشكل حدسي. (2) كلمات نية المستخدم——كيف يعبر المستخدمون عن احتياجاتهم. (3) مصطلحات المجال——الحفاظ على الإنجليزية وإضافة متعدد اللغات. المبدأ بسيط——ما يقوله المستخدم يجب أن يكون في الوصف. يتوفر التحرير اليدوي والبرنامج النصي الجماعي.

**التأثير:** 4 مهارات أساسية من 0% إلى 100% تطابق——بتغيير سطر واحد فقط. ارتفع معدل اكتشاف المهارات من 35% إلى 90%. ارتفع معدل المطابقة لغير الإنجليزية من 20% إلى 95%. انخفضت الشكاوى بنسبة 85%.

**تثبيت بسطر واحد:** `mkdir -p skills/skills-triggering && curl -sSL https://raw.githubusercontent.com/Bryan-cmf/agentic-infrastructure/main/skills-triggering/SKILL.md -o skills/skills-triggering/SKILL.md`

### 🇮🇳 हिन्दी

**समस्या:** आपके AI एजेंट में 200+ स्किल्स इंस्टॉल हैं, लेकिन जब आप हिंदी में "मेरे लिए एक वेबसाइट बनाओ" कहते हैं, तो एजेंट को Frontend Design स्किल के बारे में पता ही नहीं चलता। कारण सरल है——AI एजेंट प्लेटफ़ॉर्म की मैचिंग description के कीवर्ड तुलना पर निर्भर करती है, लेकिन 95% ओपन-सोर्स स्किल्स का description सिर्फ अंग्रेज़ी में है। "वेबसाइट" अंग्रेज़ी description में नहीं है → 0 मैच → स्किल बेकार। 242 OpenClaw स्किल्स के ऑडिट में पाया गया कि गैर-अंग्रेज़ी ट्रिगर सफलता दर सिर्फ 20% है। 4 कोर स्किल्स पूरी तरह "अदृश्य" थीं।

**समाधान:** तीन-स्तरीय कीवर्ड रणनीति। (1) कोर फंक्शन शब्द——उपयोगकर्ता स्किल को सबसे सहज रूप से क्या कहते हैं। (2) उपयोगकर्ता इरादा शब्द——उपयोगकर्ता अपनी ज़रूरत कैसे व्यक्त करते हैं। (3) डोमेन शब्दावली——अंग्रेज़ी बनाए रखें और बहुभाषी जोड़ें। सिद्धांत सरल है——उपयोगकर्ता जो कहता है, वह description में होना चाहिए। मैन्युअल संपादन और बैच स्क्रिप्ट दोनों उपलब्ध।

**प्रभाव:** 4 कोर स्किल्स 0% से 100% मैचिंग पर——सिर्फ एक लाइन बदलकर। कुल स्किल डिस्कवरी रेट 35%→90%। गैर-अंग्रेज़ी मैचिंग रेट 20%→95%। शिकायतों में 85% कमी।

**एक लाइन इंस्टॉल:** `mkdir -p skills/skills-triggering && curl -sSL https://raw.githubusercontent.com/Bryan-cmf/agentic-infrastructure/main/skills-triggering/SKILL.md -o skills/skills-triggering/SKILL.md`
